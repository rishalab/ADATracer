Component Overview

code2graph.py

Parses Ada source using libadalang, extracts procedures/functions/types, produces AdaNode objects with disk caching

req2nodes.py

Loads and parses requirements from text files into RequirementNode objects for graph integration

embeddings.py

Generates embeddings using sentence-transformers, computes cosine similarity, creates RELATED_TO relationships

github_integration.py

Fetches issues and commits from GitHub with local caching, extracts commit messages and changed files

derive_traceability.py

Derives deterministic relationships: CHANGES (Commit → AdaNode), IMPACTS (Issue → AdaNode), ADDRESSES (Commit → Issue)

graph_database.py

Neo4j persistence layer for nodes and relationships with constraint creation and query optimization

orchestrate.py

End-to-end batch pipeline that builds the complete traceability graph from source to persistence

repl.py

Interactive CLI with hybrid scoring and commands: req, reqfile, show, commits, issues

evaluate.py

Evaluates predictions vs ground truth with precision, recall, F1, and Recall@K metrics

Quick Start

Installation bash 
# Build and start the containers
docker compose build
docker compose up -d neo4j
Batch Processing bash 
# Run the complete traceability pipeline
docker compose run --rm adatracer python orchestrate.py
Interactive Exploration bash 
# Launch the REPL for interactive queries
docker compose run --rm adatracer python repl.py

# Example REPL commands:
adatracer> req "User authentication shall support multi-factor"
adatracer> show node_id_123
adatracer> commits node_id_123
Evaluation bash 
# Validate against ground truth
python evaluate.py \
  --ground-truth ground_truth.json \
  --predictions predictions.json \
  --top-k 10 \
  --output results.json

Graph Relationships

Relationship From To Description
RELATED_TO Requirement AdaNode Semantic/keyword similarity match between requirement and code element
CHANGES Commit AdaNode Commit modified the code element
IMPACTS Issue AdaNode Issue affects the code element
ADDRESSES Commit Issue Commit resolves or references the issue
SATISFIES AdaNode Requirement Code implements the requirement

Design Principles

🎲 Deterministic

Same inputs produce identical outputs. No random ML behavior or unpredictable results. Every traceability decision is reproducible and verifiable.

🔬 Explainable

Each traceability link traces back to specific evidence: similarity scores, keyword matches, or file changes. No black-box predictions.

🔧 Extensible

Modular architecture allows easy addition of new data sources, scoring methods, or analysis techniques without disrupting the core pipeline.
